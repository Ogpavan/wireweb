FAQ

Common questions,
honest answers

Is WireWeb actually free? What's the catch?
Yes, WireWeb is genuinely free for individual use. We charge for advanced features like multi-number management, analytics, and campaign scheduling in our Pro plan. The core API — send and receive messages with full webhook support — will always be free. No hidden fees, no credit card required.
Is this the official WhatsApp Business API?
WireWeb is not Meta's official WhatsApp Business API. It works by linking your WhatsApp number to our gateway via a QR code (the same method as WhatsApp Web). This means instant setup with no approval required, but your phone must remain connected to the internet. For enterprise-scale deployments with full Meta compliance, we also offer a cloud API bridge — contact us for details.
Do I need to keep my phone on for messages to send?
Yes, your phone needs to be connected to the internet for the WhatsApp session to remain active. This is a limitation of the QR-based approach. For server-side persistent sessions, our Pro plan includes a cloud device option that keeps your session alive 24/7 without requiring your phone to be on.
Will my WhatsApp account get banned?
WireWeb is designed to mimic human behavior at the API layer, significantly reducing ban risk. We recommend using it for transactional messaging (order updates, OTPs, support replies) rather than mass-marketing blasts to cold lists. Sending spam is against WhatsApp's Terms of Service regardless of what API you use.
Can I receive messages and not just send them?
Absolutely. WireWeb supports full bidirectional messaging. Configure a webhook URL in your dashboard and we'll POST every incoming message, image, document, and status update to your server in real time. Build chatbots, support systems, or anything else that needs to receive WhatsApp messages.
Which programming languages are supported?
WireWeb is a standard REST API so it works with any language that can make HTTP requests. We maintain official SDKs for Node.js, Python, and PHP. The API follows standard conventions with JSON request/response bodies and HTTP status codes, making it trivial to integrate in any language or framework.
