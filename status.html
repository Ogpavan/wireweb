WireWeb status and reliability

This page gives developers and buyers a clear place to understand WireWeb operational signals: message IDs, webhook checks, retry-safe handling, and crawlable site assets.

Operational signals to monitor

Message IDs

Store each returned message ID so support and developers can track individual send attempts.

Webhook response codes

Return fast 2xx responses for accepted events and log non-2xx responses for retry review.

Session health

Watch for session events and reconnect the WhatsApp session if the connected phone needs attention.

Crawl assets

Sitemap, robots.txt, and favicon are available for verification.

Support path

Use the contact page for deployment, onboarding, and reliability questions that need human review.

Integration checks

Test the developer, webhook, CRM, n8n, Zapier, Make, and Shopify pages before running production flows.
